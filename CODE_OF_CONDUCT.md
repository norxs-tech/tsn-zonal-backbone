# Code of Conduct

## Our Standard

The norxs Deterministic TSN Zonal Backbone is safety-critical engineering.
We hold the people in this project to the same standard we hold the code:
rigorous, honest, and respectful.

All participants — maintainers, contributors, reporters — are expected to:

- **Be professional.** Critique code and evidence, never people.
- **Be precise.** Safety engineering depends on accurate, reproducible claims.
  Do not overstate test results, coverage, or compliance status.
- **Be honest about defects.** Reporting a bug — including your own — is a
  contribution, never a fault. This project's culture treats every documented
  defect (see `CHANGELOG.md` and `docs/test_reports/`) as evidence the process
  works.
- **Respect responsible disclosure.** Security issues go through
  [`SECURITY.md`](SECURITY.md), never public threads.
- **Harassment, discrimination, and personal attacks are not tolerated** in any
  project space.

## Enforcement

Violations may be reported confidentially to **contact@norxs.com** with the
subject `[CONDUCT]`. Reports are reviewed by the project leadership; outcomes
range from a private warning to permanent exclusion from project spaces.
Reporters' identities are protected.
